# \<최상위 헤더\>

`<최상위 헤더>`를 이 문서의 전체 주제에 대한 명확하고 간결한 설명으로 교체하세요.

예시:
- "VPC 구성"
- "Pod 보안"
- "컨트롤 플레인"

## \<두 번째 수준 헤더\>

`<두 번째 수준 헤더>`를 이 주제에 대한 명확하고 간결한 설명으로 교체하세요.

예시:
- "IP 고려 사항"
- "코드형 정책(PAC)"
- "컨트롤 플레인 메트릭 모니터링"

필요한 경우 여기에 배경 정보를 추가하세요. 예를 들어:
"Kubernetes는 Kubernetes 네트워크 모델을 구현하기 위해 CNI 사양과 플러그인을 지원합니다..."

### \<세 번째 수준 헤더\>

`<세 번째 수준 헤더>`를 모범 사례 또는 문제에 대한 명확하고 간결한 설명으로 교체하세요.

예시:
- "보조 CIDR 사용 고려"
- "AWS 리소스에 대한 최소 권한 액세스 적용"
- "장기 실행 인스턴스 교체"

문제가 무엇인지, 왜 발생하는지, 문제의 증상과 같은 배경 정보를 작성하세요.

예시: "여러 연결된 VPC 또는 사이트에 걸쳐 있는 네트워크로 작업하는 경우 라우팅 가능한 주소 공간이 제한될 수 있습니다. 예를 들어, VPC가 아래와 같이 작은 서브넷으로 제한될 수 있습니다. 이 VPC에서는 CNI 구성을 조정하지 않고는 m5.16xlarge 노드를 하나 이상 실행할 수 없습니다."

# EKS 기반 스트리밍 플랫폼

:::info

**참고**: 스트리밍 플랫폼 블루프린트는 현재 개발 중입니다.
배포 예제가 저장소에 추가되면 문서가 업데이트될 예정입니다.

:::

Kubernetes에서 스트리밍 플랫폼을 실행하면 실시간 데이터 스트림을 처리하고 분석하려는 조직에 다양한 이점을 제공할 수 있습니다. Kubernetes는 자동 스케일링, 롤링 업데이트, 자가 복구 기능과 같은 기능을 제공하여 스트리밍 플랫폼의 고가용성과 안정성을 보장합니다. **Apache Kafka**, **Apache Flink**, **Apache Pulsar**를 포함한 여러 인기 있는 스트리밍 플랫폼이 이러한 사용 사례를 지원하기 위해 등장했습니다.

이러한 플랫폼은 컨테이너화된 환경에서 실시간 데이터 스트림을 쉽게 처리하고 분석할 수 있게 해주며, 실시간 데이터 처리, 이벤트 기반 아키텍처, 내결함성과 같은 기능을 제공합니다. Kubernetes의 강력한 기능을 활용함으로써 조직은 기본 인프라에 대한 걱정 없이 스트리밍 데이터 파이프라인 구축과 처리에 집중할 수 있습니다.
강력한 도구 생태계와 광범위한 사용 사례 지원을 통해 Kubernetes는 프로덕션 환경에서 스트리밍 플랫폼을 실행하기 위한 점점 더 인기 있는 선택지가 되고 있습니다.

- [Kafka](https://kafka.apache.org/)
- [Flink](https://github.com/apache/flink-kubernetes-operator)
